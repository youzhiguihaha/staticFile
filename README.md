# 📁 云存储文件管理系统

简单的自托管文件存储系统，部署到 Cloudflare Pages，支持上传、管理、分享文件直链。

## ✨ 功能特点

- 🔐 密码保护登录
- 📤 拖拽上传文件
- 📋 文件列表管理
- 🔗 一键复制直链（无需登录即可访问）
- ☁️ Cloudflare CDN 全球加速
- 💾 使用 KV 存储，免费额度大

---

## 🚀 一键部署

### 第一步：Fork 仓库

点击右上角 Fork 按钮，将项目复制到你的 GitHub

### 第二步：创建 Cloudflare Pages 项目

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 进入 **Workers 和 Pages** → **创建** → **Pages** → **连接到 Git**
3. 选择你 Fork 的仓库
4. 配置如下：

| 设置项 | 值 |
|--------|-----|
| 项目名称 | `static-file`（或你喜欢的名字） |
| 生产分支 | `main` |
| 构建命令 | `npm run build` |
| 构建输出目录 | `dist` |

5. 点击 **保存并部署**

### 第三步：创建 KV 命名空间

1. 左侧菜单 → **Workers 和 Pages** → **KV**
2. 点击 **创建命名空间**
3. 名称填：`FILES_KV`
4. 点击添加

### 第四步：绑定 KV 到项目

1. 进入你的 Pages 项目
2. **设置** → **Functions** → **KV 命名空间绑定**
3. 点击 **添加绑定**
4. 变量名称：`FILES_KV`
5. KV 命名空间：选择刚创建的 `FILES_KV`
6. 点击保存

### 第五步：设置登录密码

1. **设置** → **环境变量**
2. 点击 **添加变量**
3. 变量名称：`AUTH_PASSWORD`
4. 值：`你想设置的密码`
5. 点击保存

### 第六步：重新部署

1. **部署** → 找到最新的部署
2. 点击 **重试部署**

---

## ✅ 完成！

访问 `https://你的项目名.pages.dev` 即可使用！

---

## 📖 使用说明

1. 访问网站，输入密码登录
2. 拖拽文件到上传区域，或点击选择文件
3. 上传成功后，点击「复制链接」获取直链
4. 直链可以直接分享，无需登录即可访问

---

## ⚙️ 环境变量

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `AUTH_PASSWORD` | 是 | 登录密码，默认 `admin123` |

## 📝 注意事项

- 单个文件大小限制：约 25MB（KV 限制）
- 免费额度：每天 100,000 次读取，1,000 次写入
- 存储空间：1GB 免费
